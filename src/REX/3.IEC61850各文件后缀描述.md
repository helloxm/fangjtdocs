这四个术语均为 **IEC 61850 标准** 体系下的 **SCL（变电站配置描述语言）** 文件，用于智能变电站自动化系统的设备配置与互操作，均基于 XML 格式。

### 📄 SCL（System Configuration Description Language）
- **定义**：IEC 61850 标准规定的变电站配置语言，是所有 SCL 文件的总称。
- **作用**：统一描述智能电子设备（IED）、通信拓扑、变电站一次系统结构，确保不同厂商设备间的互操作性。
- **包含类型**：ICD、CID、IID、SSD、SCD 等。

### 📄 ICD（IED Capability Description）
- **定义**：IED 能力描述文件，由设备厂商提供。
- **作用**：定义 IED 的完整数据模型、逻辑节点（LN）、功能能力及可选通信参数，**不含具体设备实例名和特定通信地址**。
- **用途**：系统集成商基于 ICD 文件进行变电站整体配置，是系统设计的输入依据。

### 📄 CID（Configured IED Description）
- **定义**：已配置的 IED 描述文件。
- **作用**：由系统工具根据 SCD 文件与具体 IED 信息生成，包含**特定 IED 的实例化配置**（如设备名称、通信地址、数据集、GOOSE/MMS 订阅等）。
- **用途**：直接下载到 IED 完成配置，是设备投运的核心文件。

### 📄 IID（Instantiated IED Description）
- **定义**：实例化 IED 描述文件。
- **作用**：由 IED 配置工具在设备本地生成，包含**已完成本地配置的 IED 完整信息**，可直接导入系统配置工具（SCT）。
- **用途**：适用于“自下而上”的工程流程（先配置单个设备，再整合到系统）。

### ⚙️ 核心区别与工程流程
| 文件类型 | 核心特点 | 生成方 | 工程阶段 |
| :--- | :--- | :--- | :--- |
| **ICD** | 仅描述能力，无实例 | 厂商 | 系统设计 |
| **CID** | 系统级配置后的实例 | 系统工具 | 集成配置 |
| **IID** | 设备本地配置后的实例 | IED 工具 | 设备配置 |
| **SCD** | 全站统一配置（由 ICD+SSD 合成） | 系统集成商 | 最终投运 |

### 💡 工程师视角的关键提示
- **SCL 家族**：ICD、CID、IID 均基于 SCL 标准，核心差异在于**配置范围与实例化程度**。
- **CID 与 IID**：CID 是“系统视角”的配置，IID 是“设备视角”的配置，最终均需通过 CID 或二进制文件下载到设备。
- **工程实践**：优先使用 CID 文件进行设备下载，确保配置与全站系统一致；IID 适用于设备本地调试后再导入系统的场景。
